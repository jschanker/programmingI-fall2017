Introduction to Loops and Recursion

Sometimes the number of times you want to perform an operation will depend on the input.  For example, suppose you wanted to define a function that takes a positive integer n as an input and outputs the sum of the integers from 1 to n.  Forgetting that there's a general formula, and doing this directly, you would have trouble:

let sumOfPositiveIntegersFrom1To = n => {
  if(n === 1) return 1;
  else if(n === 2) return 1 + 2;
  else if(n === 3) return 1 + 2 + 3;
  else if(n === 4) return 1 + 2 + 3 + 4;
  else if(n === 5) return 1 + 2 + 3 + 4 + 5;
  :
  :
};

You can't handle every integer that's entered and the number of additions you would need would depend on n; for 3, you would perform 2 additions and for 5, you would use 4 additions. In general, for n, you would use n-1 additions.

Here is a step-by-step description of a fairly general method you could use that will be added to later for more complex problems:

  1. Draw a table with a column for each of your inputs and one for the output output. Here, we have one input n so we'd have two columns: one for the input (n) and one for the output (sum of the integers from 1 to n).
    n (input) sum from 1 to n (output)
  2. In the first row, include the starting input (base case) in which no calculations are required and the corresponding output. In this case, you would have 1 for the input and 1 for the output since the sum of the integers from 1 to 1 is just 1, no additions required.
    n (input) sum from 1 to n (output)
    1 1
  3. Fill in each subsequent row with the "next" input and corresponding output until you discover the pattern for how you update the input and output accordingly.
    n (input) sum from 1 to n (output)
    1 1
    2 3
    3 6
    4 10
    5 15

Backward Recursion:

Using step 2, introduce an if statement of the form: 

if(input variable === row 1 input) return output in row 1;

let sumOfPositiveIntegersFrom1To = n => {
  if(n === 1) return 1;
  ...
};

Using step 3, determine how to go "backwards" through the table from row to row for the inputs (call this backwardsInput) and how to go "forward" for the output (call this forwardOutput).  Use this for your else statement:

else return downOutput(   f(upInput(input variables))  ,  input variables );

For this example, we'd have:

let backwardsInput = n => n - 1; // inputs decrease by 1 going backward
let downOutput = (outputFromf, n) => outputFromF + n; // outputs increase by n going forward
