String functions & Input Processing

Quick Overview: In this assignment, you will be implementing some of Excel's Text functions, listed at https://support.office.com/en-US/article/Text-functions-reference-CCCD86AD-547D-4EA9-A065-7BB697C2A56E .  You will also be implementing functions that will be used for determining the necessary information for calculating the values of user-inputted formulas such as =QUOTIENT(14,5), by determining the name of the function (QUOTIENT in this example) and the arguments passed to it (14 and 5).  You will also be introduced to Jasmine, which you will use to create tests or specs for each of the functions you write.  Finally, you will continue to get experience with JSDoc as you document each of the functions you define.

PART A: Installing/Using Jasmine

  1. Download the latest version of Jasmine (2.8.0 as of the time this assignment is being created) from https://github.com/jasmine/jasmine/releases and uncompress it.
  2. Open the jasmine-standalone-2.8.0 directory (or whatever version you downloaded). Inside that directory, you should see SpecRunner.html. This will be the web page where the results of your tests appear. First, open it up in your web browser, and you should see that all 5 tests passed with a short description of each one (5 specs, 0 failures).
  3. Next, right mouse click the file SpecRunner.html and select Open With. Select a text editor such as TextEditor on Mac or Notepad on Windows. You can also open the file with a text editor that'll highlight as in repl.it such as Sublime Text, which you can download at https://www.sublimetext.com . After you've opened the file, add the following line immediately below <script src="src/Song.js"></script>: 
      <script src="src/stringFunctions.js"></script>
    This will be the source file where you define all of your string functions. Also, below the line <script src="src/PlayerSpec.js"></script>:, add the following: 
      <script src="spec/stringFunctionsSpec.js"></script>
    This will be the file where you create your suite of tests.  Be sure to save SpecRunner.html after adding these lines.
  4. Create a new file and save it as stringFunctions.js in the src directory.  Be sure to save this file in the src directory in jasmine-standalone-2.8.0.  Then define a function len that takes a string text as an input and returns its length.
  5. Create a new file and save it as stringFunctionsSpec.js in the spec directory.  Be sure to save this file in the spec directory in jasmine-standalone-2.8.0.  Then enter the following code: 
    describe("String functions", function() {
  it("returns the length of a string correctly", function() {
    expect(len("")).toBe(0);
    expect(len("abc")).toBe(3);
    expect(len("Foo 123!")).toBe(8);
  });
  it("title for next test", function() {
    expect(2+2).toBe(4);
  });
  it("title for next next test", function() {
    expect(2+2).not.toBe(3);
  });
});
    This tests the len function you just defined with the strings "", "abc", and "Foo 123!", Notice how each expectation for the first length test is written: expect(__________) with the __________ replaced by the function call for the input to test followed by .toBe(___) with the ___ replaced by the expected output. The expectations are all wrapped inside a so-called anonymous function (function with no name) which is the second argument to the it function. The first argument to it is a string representing what this spec or test is supposed to demonstrate (for the first one, it "returns the length of a string correctly", indicating that your len function seems to be working properly.) There are other tests for you to fill in, which are each wrapped in an outer anonymous function, which is the second argument to the describe function.  The first argument is a description for the suite of tests (in this case "String functions" indicating that your string functions are being tested.
  6. After you have completed these steps, open up SpecRunner.html in the browser again (or reload it if it's still open). If you implemented your length function correctly, you should now see 8 specs, 0 failures.

PART B: Implementing Excel Text Functions

In this part, you will be implementing parts of Excel's text functions (parts for now: e.g., CONCATENATE can take more than 2 inputs to join together but the function you'll define will only work with 2 for now).  You should write comments before each function in JSDoc format.  You should also test each function in Excel and add tests in Jasmine with the expected output to see if the functions you defined might work correctly.

LEN: This was completed in Part A.  You defined a function len that takes a string text as an input and returns the number of characters in the string to emulate Excel's LEN. See: https://support.office.com/en-US/article/LEN-LENB-functions-29236f94-cedc-429d-affd-b5e33d2c67cb 

CONCATENATE: Define a function concatenate(text1, text2) that takes two strings text1 and text2 as inputs and returns the string in which they are joined together (e.g., for text1 = "co" and text2 = "der", it should return "coder") See: https://support.office.com/en-US/article/CONCATENATE-function-8f8ae884-2ca8-4f7a-b093-75d702bea31d 

LEFT: Define a function left that takes a string text and whole number numChars as inputs and returns the first numChars of text. See: https://support.office.com/en-US/article/LEFT-LEFTB-functions-9203d2d2-7960-479b-84c6-1ea52b99640c 

RIGHT: Define a function right that takes a string text and whole number numChars as inputs and returns the last numChars of text.   See: https://support.office.com/en-US/article/RIGHT-RIGHTB-functions-240267ee-9afa-4639-a02b-f19e1786cf2f 

FIND: Define a function find that takes a string findText and a string withinText and a positive integer startNum as inputs and returns the position of the beginning of the first occurrence of findText within the text withinText, starting at startNum  numChars of text.  (Here, positions start at 1, not 0.)  See: https://support.office.com/en-US/article/FIND-FINDB-functions-c7912941-af2a-4bdf-a553-d0d89b0a0628 

SEARCH: Define a function search that takes a string findText and a string withinText and a positive integer startNum as inputs and returns the position of the beginning of the first occurrence of findText within the text withinText, starting at startNum numChars of text.  (Here, positions start at 1, not 0.)   Note: The difference between this function and FIND is that this one ignores case (e,g., SEARCH("cODeR", "Top Coder!", 1) is 5. (HINT: Consider using toLowerCase in your answer : https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/toLowerCase .)  https://support.office.com/en-US/article/SEARCH-SEARCHB-functions-9ab04538-0e55-4719-a72e-b6f54513b495 

MID: Define a function mid that takes a string text, a positive integer startNum and a whole number numChars as inputs and returns the substring of characters of text, starting at position startNum , that's numChars many characters long.  (Here, positions start at 1, not 0.)  See: https://support.office.com/en-US/article/MID-MIDB-functions-d5f9e25c-d7d6-472e-b568-4ecb12433028 

REPLACE: Define a function replace that takes a string oldText, a positive integer startNum, a positive integer numChars, and a string newText as inputs and outputs oldText with the first numChars characters from startNum replaced with the string newText.  See: https://support.office.com/en-us/article/REPLACE-REPLACEB-functions-8d799074-2425-4a8a-84bc-82472868878a

SUBSTITUTE: Define a function substituteOnce that takes three strings, text, oldText, and newText, and outputs text with the first occurrence of oldText replaced with the string newText.  Now try implementing substitute which takes text, oldText, and newText, and a whole number instanceNum that returns text with its first instanceNum occurrences of oldText replaced by newText. Think whether or not you can use substituteOnce to do this. See: https://support.office.com/en-us/article/SUBSTITUTE-function-6434944e-a904-4336-a9b0-1e58df3bc332 

PART C: Beginning of Input Processing

getFunctionName: Define a function that given a string input for an Excel spreadsheet function, returns its name (e.g., for an input of "QUOTIENT(14,5)", getFunctionName should return the string "QUOTIENT").

removeArgs : Define a function that given a string input of a list of arguments args separated by commas and a positive integer num removes the first num arguments. You may assume that your input has more arguments than the argument number you're getting.  For example, for args = "\"foo\",12345,\"bar\",\"molloy\",\"A2\"", and num = 3, removeArgs should return "\"molloy\",\"A2\"".  Note: the slashes before the quotes indicate that they should literally be interpreted as quotation marks that are part of the string and not the end of the string.

getArgNum:Define a function that given a string input of a list of arguments args separated by commas and a positive integer num , gets argument number num .  For example, for "\"foo\",12345,\"bar\",\"molloy\",\"A2\"", getArgNum(1) returns "foo", getArgNum(2) returns 12345, getArgNum(3) returns "bar", and getArgNum(4) returns "molloy".  For each function, you may assume your input has more arguments than the argument number you're getting.  (E.g., getArg3 would only be called for a comma-separated list of 4 or more arguments.)

getFinalArg: Define a function that given a string input of a list of arguments, gets the final argument in the list (e.g., "A2" for the previous input)

removeEnds: Define a function that given a string as an input removes its first and last character; this will be for e.g., stripping the quotes off of string literals (e.g., for "foo", it would return foo).
