Day 4 Notes: Boolean-Valued Functions

Boolean-Valued Functions

Boolean Variables and Functions

  • A Boolean literal is either true or false. Compare this to a number literal such as 3.14 or a string literal such as "Hey".
  • You can assign Booleans to variables just as you can numbers or strings. Boolean variables (or flags) often have names starting with is, are, am, etc. 
    let isTuesday = false;
let amTired = true;
let areResultCellsReadonly = true;
  • Functions can accept Booleans (or numbers, strings, etc.) as inputs. A function that returns a Boolean as an output is a Boolean-valued function.

Boolean Operators

Just as there are operators on numbers and strings (e.g., +, *, etc.), there are operators on Booleans as well. (Think logic and truth tables from high school.). In JavaScript (and Java, C#, C++, Ruby, etc.) you can use && for logical and, || for logical (inclusive) or, and ! for logical not.

p q !p p && q p || q
true true false true true
true false false false true
false true true false true
false false true false false

Sometimes, you want to directly specify the outputs for the inputs you're given and using a single logical operator from &&, ||, and ! won't produce the outputs you want.  However, by using a combination of and's, or's and not's, you can get the desired output.  If you've taken propositional logic before, you may have learned for example that  ~(~p ∧ ~q) is logically equivalent to p ∨ q (∧ is for and, ~ for not, and ∨ for inclusive or), meaning that for any possible pair of truth values for p and q (TT, TF, FT, and FF), ~(~p ∧ ~q) and p ∨ q will match (either both true or both false).  This means that you don't need to use || at all; you can replace every occurrence of  p || q with !(!p && !q). Here's a general procedure you can use to define a logical operator in terms of and's, or's and not's:

  1. Draw a truth table with the inputs and the outputs you want them to produce.
  2. Identify all of the rows with true outputs, starting with the first.
  3. For each such row, proceed from left to right through the letters. If it's T, write the letter whereas if it's false, write ! and then the letter. Use && between each letter or its negation as you proceed through the columns.
  4. When you've finished the previous step for a given row, add an || at the end and repeat the previous step for the next row with a true output (provided you're not already at the last such row).

De Morgan's Laws

Just as there are laws expressing numerical equivalences such as the distributive property (i.e., x * (y + z) is equivalent to x*y + x*z), there are laws expressing equivalences of Boolean expressions. De Morgan’s Laws are very important in programming!

// The following Boolean expressions are logically equivalent:
     !(p && q);
     !p || !q;
// i.e., not (p and q) is logically equivalent to not p or not q;
// if it's not the case that both p and q are true, then at least one of p or q must be false
// The following Boolean expressions are logically equivalent:
     !(p || q);
     !p && !q;
// i.e., not (p or q) is logically equivalent to not p and not q;
// if neither p nor q is true, then BOTH p and q must be false

If we take the opposite truth values of the first example, we would get the following

p (input 1) q (input 2) OPPOSITE of Desired output of mysteryOperator
true true false
true false false
false true true
false false false

To produce this output, we would look only at the third row and get the following:

p && !q;

Since it's the opposite of what we want, we can write our original solution in the must more compact form as follows:

!(p && !q);
// By De Morgan's Laws, we could use the following:
!p || !!q;
// Since !!q is logically equivalent to q (opposite of the opposite truth value of q is the truth value of q),
// we could write this as follows:
!p || q;

Shortening Boolean Expressions

You can potentially shorten your solutions by defining operators in terms of previously defined ones.  For example, from propositional logic, we know that p ↔ q is logically equivalent to p → q ∧ q → p so had we defined cond(p, q) first, we could define nxor(p, q) as follows:

let and = (p, q)  => p && q;
let nxor = (p, q) => and(cond(p, q), cond(q, p));

Can you think of an even shorter way (the hint is in the name of the function)?  More generally, just as you would simplify algebraic expressions using rules of algebra, by using rules of logic, you could shorten logic expressions.  Consider the solution to the first example:

p && q || p && !q || !p && !q;
// (p && q || p && !q) is logically equivalent to p:
// Either q or not q must be true so if p is true, then either (p and q) or (p and not q) will be 
// true as well.
// In the other direction, if (p and q) or (p and not q) is true, then in both cases p must be 
// true. So by substitution, the above expression is logically equivalent to:
p || (!p && !q);
// "Distributing" the p in, we get the following logically equivalent expression:
(p || !p) && (p || !q);
// p or not p is always true so we get the following logically equivalent expression:
true && (p || !q);
// true behaves as the identity for and just as 1 behaves as the identity for multiplication;
// that is 1 * x = x * 1 = x (you get back what you started with) so the above is logically
// equivalent to the following:
p || !q; // This agrees with the answer in the previous section

In digital circuit design, you would use use Karnaugh maps to shorten long series of ands, ors and nots.

Comparison Operators

A Boolean can result from a comparison operator.  Here are some of JavaScript's comparison operators:

=== : Think "really identically" equal; one = is for assignment; == also checks for "equality" but not in the way you might expect.  This is due to interesting conversions in JavaScript that are made when comparing between different types such as numbers and strings using ==.  For example, 0 is considered equal to the empty string so 0 == "" evaluates to true, but no such conversions are made when using === so 0 === "" evaluates to false.

console.log(0 === 1); // logs false

!==: Not equal

console.log(0 !== 1); // logs true
console.log(2 !== 2); // logs false

<, >: Less than, greater than

console.log(2 > 2); // logs false
console.log(1 < 2); // logs true

<=, >=: Less than or equal to, greater than or equal to

console.log(2 >= 2); // 2 ≥ 2: logs true
console.log(5 <= 2); // 5 ≤ 2: logs false

Here are some examples of functions that take numbers or strings as inputs and return Booleans as outputs:

let isItTuesday = day => day === "Tuesday"; // returns true exactly when day is Tuesday
let isPositive = num => num > 0; // returns true exactly when num is positive
// Note: num > 0 || === 0 would NOT work since === 0 alone (without the num) is incomplete;
// Need statement that is either true or false on both sides of a binary logic operator
let is nonNegative = num => num > 0 || num === 0; // Could also use num >= 0;

Here are some examples where the functions above are called:

console.log(isItTuesday("Wednesday")); // logs false
console.log(isItTuesday("Tuesday")); // logs true
console.log(isPositive(0)); // logs false
console.log(isPositive(-1.2345)); // logs false
console.log(isPositive(3.14159)); // logs true
