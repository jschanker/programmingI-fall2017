Statements, Output, and Comments to the Reader

You can print messages by using console.log.  You can separate (print) statements with semicolons.

console.log("Hello World!");
console.log("The meaning of life is 42.");

A message to be printed as is needs to be placed in between a pair of double quotes so that it's not interpreted as a variable name (as described in the following section):

// The following would be an error if // is removed on the next line: ReferenceError: bar is not defined:
// console.log(bar);

Removing the // means that the computer will try to print the value of the variable bar instead of the word, bar. The // tells the computer to ignore what follows on the line as it is for human use only.

Lines starting with // are known as single line comments. They can be used as notes which can provide context for your code. Comments are optional but useful; removing them will have no effect on how your program runs.

const a = 9.8; // a is approximate acceleration due to gravity in meters per second

Variables

Use a variable to name something stored in memory so that you can refer to it later.

  • Use descriptive variable names in camel case: e.g., firstName, bestGameScore
  • Assign variables with let statements: 
    let x = 7;              // x gets assigned 7
let y = x + 5;          // y gets assigned 5 more than x (12)
let name = "Jane Doe";  // variables can be assigned words
  • Assign with const for constants. Constants are sometimes expressed in all CAPS: 
    const PI = 3.141592653589793;
  • You can print the values of variables just as you do text using console.log, but they need to be placed outside of the quotes so that they're not interpreted literally: 
    console.log("x =", x, "and y =", y);
console.log("Hi,", name, ".");
console.log("PI =", PI);

Math Operators

You can use math operators with specific numbers or variables that refer to numbers:

  • + : Addition
  • - : Subtraction
  • * : Multiplication
  • / : Division
  • Math.trunc(# / #) : Integer Division (do the division and remove the decimal point and all digits that come after it; truncate it) - See Math.trunc()
  • % : Remainder
  • **: Exponentiation (NOT ^, which is bitwise XOR): Note ** has not been standardized yet and will therefore not work across all browsers.
  • Operations performed in PEMDAS order: (): Parentheses first; then Exponentiation (**), then Multiplication/Division (*, /, and % for remainder), then Addition/Subtraction (+/-).
//                                               _2_R4
console.log(Math.trunc(14 / 5)); // prints 2 : 5|14
console.log(14 % 5);             // prints remainder of 4

Defining/Calling Functions

You can use variables to refer to functions as well:

  • Define functions using arrow notation: 
    let double = x => 2*x;
  • The variable double refers to a function that takes x as an input and outputs two times it.
  • The variable x in the above expression is called a parameter variable. It is a placeholder for the input.
  • 2*x is the return value or output of the function.
  • Note the x can be replaced with other variable names such as t or even foo. In those cases, you would use t => 2*t and foo => 2*foo, respectively.
  • Evaluate functions at specific inputs by writing them as you would in your PreCalculus class. Use console.log to display the result: 
    console.log(double(0));     // outputs 0 : 0 => 2*0
console.log(double(5));     // outputs 10 : 5 => 2*5
console.log(double(-1.57)); // outputs -3.14 : -1.57 => 2*-1.57
  • The inputs 0, 5, and -1.57 are called arguments and are said to be passed to the double function.  To get the output, we substitute the argument for the parameter variable x and calculate what's to the right of the arrow. (E.g., for 5, we substitute 5 for x and calculate 2*5, which is 10.)
  • Functions that perform a computation such as double without changing anything are known as pure functions. console.log is a function but not a pure one. It does change something, mainly the console where you will see output displayed. When a function changes the state of something, it is said to have side effects. A pure function by definition has no side effects. If we wrote double(5); without using console.log, we would not notice that any computation was done.
  • Functions can have more than one input. In these cases, you must use parentheses to group the parameter variables together and a comma in between them to separate them. 
    let rectangleArea = (length, width) => length * width;
console.log(rectangleArea(5, 2)); // logs 10 (5 * 2)
  • IMPORTANT NOTE: The * is required so you couldn't for example write lengthwidth or the computer would get confused. Did you mean one long variable lengthwidth? More generally, there is no implicit multiplication; to calculate 2x, you must explicitly include the * symbol so that would be 2*x.

Defining functions without formulas

Pure functions like the distance function you defined for the above exercise can represent formulas.  You plug in or substitute the values for the input variable(s), which is the time t in this case, to determine the output, which is the distance d in this case.  The formula or function should work for all allowed inputs, not just some of them.  

For some functions you want to define, the formula is not given so you need to determine what the formula is.  And again, it needs to work for all allowed inputs.  To do this, it is often immensely useful to calculate the desired outputs for some inputs to determine what operations you need to perform.  When selecting your inputs, you need to be careful to select a sample of different types of inputs to make sure your calculations aren't specific to certain cases.  Let's consider an example to illustrate:

Example (Pairs of socks): Define a function numOfPairs that takes a number of socks as an input and returns the number of whole pairs.  (It's okay to pair socks that don't match; we'll make a new style that way!)

Solution: Let's form a table showing some inputs and the corresponding desired outputs.

socks (input) Output from numOfPairs
0 0
2 1
8 4
18 9
50 25

From the table, we might conclude that the formula is given by the following:

pairs = socks / 2;

Specifically, we seem to divide the number of socks by 2 to get the number of pairs. But is this correct for every possible number of socks we might have?  It seems we may have overlooked the case in which we had an odd number of socks.  For example, if we plug in 5 for socks, we'll get that we have 2.5 pairs, but we should get 2 since we wanted the number of whole pairs.  Adding some odd numbered inputs to our table, we'd have the following:

socks (input) Output from numOfPairs
0 0
1 0 (old formula: 0.5)
2 1
5 2 (old formula: 2.5)
8 4
18 9
31 15 (old formula: 7.5)
50 25

From the table, it still seems that we can use the same formula as before, dividing the number of socks by 2, so long as we remove the .5 to the right of the decimal place after the division should it be present.  The desired function definition, reflecting the new formula, would seem to be as follows:

let numOfPairs = socks => Math.trunc(socks / 2);

We need to now make sure that this new function definition works for the original even numbered of socks.  It does since truncating the result of the division by 2 has no effect on whole numbers such as 4 (e.g.,  Math.trunc(8/2) = Math.trunc(4) = 4).  So we now have our desired function definition.  NOTE: A more natural interpretation of this function definition is integer division by 2, where we ignore the remainder.
